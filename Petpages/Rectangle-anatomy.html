1.
How to Read CSS

Why CSS?

CSS (Cascading Style Sheets) is a collection of rules for your page.

HTML is the content of the page.
CSS is the style of the page.

The rules are structured the same way every time.

Below, I pick the header of the page and say "Your background color will be: lavender."

You pick one element of the page, and you put all the rules for that element inside {curly braces}. You separate each rule with a semicolon.

Because the curly braces collect all the rules, you can list many rules for one element all inside them. Make sure that all rules, even the LAST one, end in a semicolon.



In CSS terminology, each of those parts is:

Let's break down each color-coded part of the structure:

  • Selector: Tag that identifies which content your rule affects.
  • Brackets: The curly braces that surround each selector's ruleset.
  • Property: One characteristic, like "color" or "width"
  • Value: The decision you made about the property, like "red" or "500px"
  • Semicolon: Ends each property rule, including the last one before the curly brace.

Selector Types

Without getting into fancy stuff, there are three main types of selectors: elements, classes, and IDs. In increasing specificity.

Element
example: div, header
Generic - like all divs or all links.

Class
example: .navbar, .gothParagraph
Versatile. Classes create categories for more efficient styling.

ID
example: #bibliography
Give ONE element its own unique ID and then call ONLY that element to style.

You can also combine selectors.

I don't want to get too complicated with this, but it's important to know for some common page styling - you can get more specific by using multiple selectors, or referring to elements with a specific relationship.

Some of our code won't work on everything. There are a few elements which have really specific selectors that we need to override.

Note: In CSS, the more specific rule wins.
    ID > class > element-type
    #bigdiv > .bigdiv > div

    In a tie, the later rule in the code document wins.
