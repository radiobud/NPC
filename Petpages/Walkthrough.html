0. CSS and HTML

An important structure reminder about CSS: You will always be editing NPC pages in an HTML context, so you need to put the CSS inside html tags.

I make a habit of starting any new code with style tags:

<style>

</style>

Then put ALL your CSS inside them. Keep any other content (adoptables, description of your pet, etc) outside and after those style tags.

<style>
    body{
        background-color: white;
    }
</style>

<!--This is a comment. Below is regular HTML-->

<div>
    Hi! I'm Buddy! Welcome to my page.
</div>

<img src="\example.png">

1. Remove NPC Extras

To make the page look simple, we'll remove the headers, sidebar, and footer. I think it's a good idea to leave some sort of navigation, but we'll save that for the bonus section.

Here are the NPC basics.

The code to remove the header etc looks like this:

display: none;

/*Step 1*/

.hdr, #hdr, #mb, .navhover, .sidebar-container, .footertext, #retrotopbar, .results {
    display:none;}

You can also remove the horizontal line, which is an HTML element called hr.

2. Change background color and font

Next we can doll up the page a little.

Change colors

    CSS properties:
  • background-color: changes, well, bg color
  • color: changes font color
  • font-family: changes typeface
  • font-weight: changes bold/light font

Let's make the following changes:

  • background-color: lavender;
  • color: navy;
  • font-family: "Garamond", serif;
  • font-weight: bold;

Note: Backup generic fonts are a good idea. Separate with commas. The browser will pick fonts down the list until it hits one it knows.

The selector for all these properties is not obvious. I'll put them on the #sfp_body element for now, which is a pretty high-level container for most content on the petpage.

The code below might start to look more complicated, but look - it's just the list from above, placed in between the curly braces.

color + font styles

/*Step 2*/

#sfp_body{
    background-color: lavender;
    color: navy;
    font-family: "Garamond", serif;
    font-weight: bold;
    }

By now your petpage should look something like this (I'm leaving my pet links and BD stats so there's more to look at). The color and font-family properties didn't do anything yet; that's because the main text has more specific selectors we need to deal with later.

Not a fan of that empty space and alignment? Let's style the placement next.

3. Center the Container

In CSS, "how to center a div" is a meme question - because there genuinely are many tricky aspects to this seemingly simple task.

The easiest way to center something is to give it a margin of auto. Auto-margin means that the margin will TRY to be as BIG as it can be. Since both left and right sides are given the auto-margin, they split the entire margin in half and take up as much as they can on their half. At least, that's how Josh Comeau explains it!

Centering on lookups was a little weird for me to figure out. We'll use a relationship between elements to select the right thing.

We're using the plus (+) selector: A + B
Which means: Select the B after any A. (The last thing listed in the selector is what is being altered.)

map + table

/*Step 3*/

map + table {
    margin: auto;}

Let's go ahead and put a different background color on the pet container so you can see it better.

.petlookupcontainer{
    background: lightblue;
    }

Try it on your own pet lookup! You should have a nice, centered container now.

4. Add your own content

Let's start adding some of your own words and content to the page.

You may want to put your description inside div elements, so that you have even more control over their look.

Let's write the HTML for that div first. We are going to give it a class called "desc" (for description).

div in HTML

<div class="desc">    
    Are you having fun coding?
</div>

Now we can style the description div. I'll put a lot of properties here; feel free to play around with numbers or colors.

Note: You must set a width on the div for margin:auto; to have any effect. Otherwise the div will default to being as wide as it can be.

Example div styling

/*Step 4*/

.desc{
    background-color: goldenrod;
    color: purple;
    font-family: Garamond, serif;
    font-size: 1.5rem;
    border: 2px dotted hotpink;
    width: 300px;
    margin: auto;}

I've set the whole petlookupcontainer to a blue background, too, so it's a little cleaner.

.petlookupcontainer{
    background: lightblue;}

Pause and put in your own div with words!

5. Pretty up the Default Page

Time to take control of those page elements!

Let's take a look at the elements on a pet lookup.

The majority are classes such as .petImage (your pet's pic) and .petLinks (your Neomail and Petpage links).

First, we are going to style:

  • Pet Stats
  • Header Font
  • Background Images

Pet stats

Let's change the color and font of the stats. Remember, we need to be specific. Also, I'm using some other color values here, rgb and hex - those are more flexible than just the named colors.

/*Step 5*/

.petstatsblock{
    background-color: rgb(153, 234, 157);
    color: #97451c;
    font-family: Garamond, serif;
    }

The header font is another weird one. It's best selected with #center, although that also applies to some petpet font, so you have to finagle if you also have a petpet.

/*Step 5.1*/

#center{
    color: goldenrod;
    font-family: Garamond, serif;
    font-size: 2rem;
    }

Finally, we'll change the background image of the lookup. If your image is small, you may want to change the repeat or the size of the image. I suggest a CSS tutorial like W3Schools' for that!

/*Step 5.2*/

#sfp_body{
    background-image: url(https://images.unsplash.com/vector-1759878845510-e9ed73b7c9e0?q=80&w=1652&auto=format&fit=crop&ixlib=rb-4.1.0&ixid=M3wxMjA3fDB8MHxwaG90by1wYWdlfHx8fGVufDB8fHx8fA%3D%3D);
    }

Sometimes you have a lot of images that are hard to select, and it's easier to do a sort of find-and-replace for the image source. You need to know the current image url, and replace it with your chosen art url.

Picky Overlays

   /*Replace an exact image*/
img[src="CURRENT"] {content:url('NEW'); }

Note 1: The code only works if it is the only thing on your page with that exact image! It can also work for belongings (replacing a regular item with a fandom image etc). This code is from Revel.

Note 2: You will likely need to set an image size to make sure overlays display right:

Change Width

    /*Pet image example*/
.petImage {
    content:url('LINK');
    width: 150px;}
