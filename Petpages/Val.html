How to Read CSS

Why CSS?

CSS (Cascading Style Sheets) is a collection of rules for your page.

HTML is the content of the page.
CSS is the style of the page.

CSS is not magic - it's just giving orders.

The nice thing about CSS, in my opinion:
it is very readable.

The rules are structured the same way every time.

You pick one element of the page, and you put all the rules for that element inside {curly braces}. You separate each rule with a semicolon.

You can list many rules for one element all inside the same set of curly braces. Make sure that all rules, even the LAST one, end in a semicolon.



In CSS terminology, each of those parts is:

Let's break down each color-coded part of the structure:

  • Selector: Tag that identifies which content your rule affects.
  • Brackets: The curly braces that surround each selector's ruleset.
  • Property: One characteristic, like "color" or "width"
  • Value: The decision you made about the property, like "red" or "500px"
  • Semicolon: Ends each property rule, including the last one before the curly brace.

Takeaways

If I can give you one thing from this section, it's this:

Learn the structure of the CSS. If you are comfortable reading the structure, you will know what to look for when you want to customize someone's premade code, or how to troubleshoot. You can always look up properties and specifics, like "how to center a div" and "how to add border shadow" but you'll work more confidently with the basic foundation.

HTML + CSS

Selectors are in the HTML.

Without getting into fancy stuff, there are three common types of selectors: elements, classes, and IDs. In increasing specificity.


Element - General HTML bit. example: div, header
Class - Category you make. example: .navbar, .gothParagraph
ID - SINGLE element you name. example: #bibliography

These can also be combined. You can select all the p elements within a div, for example, like this:

div p {color: red;}

You can list multiple selectors for one ruleset. Like this:

.hdr, .navhover, .footertext {display:none;}

How to Use HTML to Your Advantage

The selectors above - plain element, or class/id names - are grabbing something that is IN the HTML. Classes and IDs are named when the HTML is written. If this is NPC code, you've got to figure out what those names are. If it's your code, then you get to name classes or IDs whatever you want. Here is an example code:

    <section>

       <div>
        <p>Hello World</p>
        <p>Wow!<p>
       </div> 

       <div>
        <p>Goodnight moon</p>
        <p>Zzzz<p>
       </div>

    </section>

When there are multiple elements that you want to style the same way, it can be nice to give them a class.

    <section>

       <div class="greeting">
        <p>Hello World</p>
        <p>Wow!<p>
       </div> 

       <div class="greeting">
        <p>Goodnight moon</p>
        <p>Zzzz<p>
       </div>
       
    </section>

What if we have more unique elements that we're doing fun things with? Then ID is appropriate.

    <section>

       <div class="greeting">
        <p>Hello World</p>
        <p id="red">Wow!<p>
       </div> 

       <div class="greeting">
        <p>Goodnight moon</p>
        <p id="blue">Zzzz<p>
       </div>
       
    </section>

You can get WAY more nuanced with selectors. I recommend the browser game CSS Diner for practice. It helps you get comfortable with all types of selectors, plus gives you a greater understanding of how CSS works with HTML.

Heads up: Style Tags

All the coding you will do on NPC is in an HTML document. The browser reads it all as HTML, and your CSS will not work unless you tell the browser, "HEY! This part is CSS."

There is an HTML tag for "hey it's CSS!" - the style element.

Everthing inside the style tag will be read as CSS.

In general, your <style> section should all be together at the top of the document.
