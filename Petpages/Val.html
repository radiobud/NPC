How to Read CSS

Why CSS?

CSS (Cascading Style Sheets) is a collection of rules.

HTML is the content of the page.
CSS is the styling.

CSS is not magic - it's just giving orders.

The nice thing about CSS, in my opinion:
it is very readable.

The rules are structured the same way every time.

Pick the header of the page and say "Your background color will be: lavender."

In CSS terminology, each of those parts is:

Let's break down each color-coded part of the structure:

  • Selector: Tag that identifies which content your rule affects.
  • Brackets: The curly braces that surround each selector's ruleset.
  • Property: One characteristic, like "color" or "width"
  • Value: Setting for the property, like "red" or "500px"
  • Semicolon: Ends each property rule, including the last one before the curly brace.

HTML + CSS

What is CSS selecting? HTML.

The selectors above are targeting something that is IN the HTML.

Without getting into fancy stuff, there are three common types of selectors: elements, classes, and IDs.

Element - General HTML chunk. example: div, header
Class - Category you make. example: .navbar, .gothParagraph
ID - SINGLE element you name. example: #bibliography

These can also be combined. You can select all the p elements within a div, for example, like this:

div p {color: red;}

You can list multiple selectors for one ruleset, separated with commas. Like this:

.hdr, .navhover, .footertext {display:none;}

How to Use HTML to Your Advantage

Classes and IDs are a part of the HTML.
If this is NPC code, you've got to figure those out in order to select the right thing. If it's your code, then you get to name your own classes or IDs. Here is some sample HTML:

    <section>

       <div>
        <p>Hello World</p>
        <p>Wow!<p>
       </div> 

       <div>
        <p>Goodnight Moon</p>
        <p>Zzzz<p>
       </div>

    </section>

When there are multiple elements that you want to style the same way, you can give them a class.

If we have more unique elements, then ID is appropriate.

    <section>

       <div class="greeting">
        <p>Hello World</p>
        <p id="red">Wow!<p>
       </div> 

       <div class="greeting">
        <p>Goodnight moon</p>
        <p id="blue">Zzzz<p>
       </div>
       
    </section>

You can get WAY more nuanced. I recommend the browser game CSS Diner for practice. It helps you get comfortable with all types of selectors, plus gives you a greater understanding of how CSS works with HTML.

Heads up: Style Tags

All the coding you will do on NPC is in an HTML document. The browser reads it all as HTML, and your CSS will not work unless you tell the browser, "HEY! This part is CSS."

There is an HTML tag for that - the style element. Everthing inside the style tag will be read as CSS.

In general, your <style> section should all be together at the top of the document.
