HTML structure

HTML is the structure and content of the page.

CSS is the decoration and, to some extent, the arrangement of that structure.

HTML can be semantic or non-semantic.

semantic
tells you what it is: <nav> <header> or <img>
non-semantic
could be anything, and it lets you structure or style the page: <div> <em> or <span>

There are two important things I'd like to share about semantic HTML: one, that it is helpful for accessibility (screen readers can navigate with it), and two, that it makes your own code easier to read. All those anonymous divs add up!

Elements and Tags

Any one HTML object is an element. Elements are surrounded (usually) by tags that define them. In the Neoboards, tags use square brackets [like this] but in regular HTML, they use angle brackets <like this>.

Tags usually surround content to define it - like:

<h1>This is a Header</h1>

Some elements are empty, meaning they do not require a second closing tag. The most common empty tags are line breaks <br> and images <img src=etc>.

Neopets Classic HTML rules

Apart from the (hopefully) obvious inappropriate content rule, there are things that NPC requires from all user code: most importantly, what it should not change.

From NPC Site Rules #11:
Covering, hiding, or altering any of your user/pet names, stats, colors, or other important information is against the rules. While image overlays are okay, they must leave the things listed above accurately depicted/fully visible to others.

I interpret this to mean that styling NPC elements like the sidebar is allowed, as long as you aren't misrepresenting what pet or user someone is looking at, or the details of that page.

Pet overlays (alternate images replacing your pet's portrait) are popular, and explicitly allowed here, so I interpret the rule to mean that if you use an overlay, you can't change or hide the text that says, for example, business boy the yellow Gelert. You can make that text look pretty or move it around the page, though.

I also think it's important to provide the general navigation that someone expects when they open a page, so if I hide the NPC sidebar, I add my own nav that leads to key places on the site. Guildenstern's lookup has an example of this alternate nav.

What does clean code mean?

Clean code in general means properly formatted and ideally readable. But clean code means one more thing to Neopia.

The box above your page editor announces All HTML and Stylesheets must be recognized as safe by our filter... so what does the filter check for?

The linked help page gives one overt instruction: elements object, form, and input are banned. If you get an error message, it also mentions that iFrames are banned.

iFrames and Object
used to embed things like webpages within webpages
input and form
can be used along with other coding to collect information from users.

A note: Can you create a purely decorative button that doesn't send data? Yes - see the one below!

CSS Format

CSS is placed between HTML style tags, which tell the browser this isn't content, it's layout.

Curly braces surround the actual commands of the CSS. They're like the angle brackets of HTML (to me, anyway). Here is the abstract structure:

<style>

     selector {
     property: value;
     }

</style>

Here is an actual example of CSS:

<style>

     header {
     background-color: lavender;
     }

</style>
<style>

     hey thing {
     look like: this;
     }
     
</style>
<style>

     hey thing {
     have font like: this;
     have a background like: this;
     be shaped like: this;
     }
     
</style>

The element is the HTML being selected for styling, which we'll get to in a second. The property is one quality of the element, like text color or width. Properties do not use spaces. The value is the choice you make about that property (e.g. be blue, or be Verdana font, or fill the screen, etc.) ...The curly braces are usually split that way for readability, but you can keep it all one line.

Important!! Semicolons!!

A semicolon follows every single line of CSS, including the last one right before the closing brace. If my CSS broke, more than half the time I just dropped a single semicolon somewhere.

Selector Types

On any site, you will want to call specific things and tell them to behave differently. On NPC, to re-style existing site elements like the standard sidebar, you need selectors.

Element
example: div {
Style every element of a generic type, like all divs or all links.
format: Just the type of element, like in the header example above.
Class
example: .big-news {
Versatile. You can give HTML elements a class to group them for more efficient styling. I do this most for different types of containers for layout, like sidebars and small blurbs.
format:Start with a period.
ID
example: #bibliography {
You can give ONE element its own unique ID and then call ONLY that element to style.
format:Start with a hash symbol (#). Same rules about spaces.

One more thing: comments in CSS look like this:

/* Here is a comment */
The browser ignores comments. You can use comments to remind yourself what a section was for... or, if you're testing code, you can put comment tags around a line to neutralize it without having to cut and re-paste it.
